# 产品升级路线图：从"到期管家"到"AI 美食生活管家"

> 目标：将工具型产品升级为"AI + 美食生活"的生活方式平台
> 关联战略：匹配开发者"美食垂类运营 + AI PM 转型"的职业定位
> **2026-07-08 16:10 审计修订时点**：v1.0 → v3.0 战略路线图，模块设计详细。

---

## 一、升级愿景

### 现在（v1.0）
**保质期管理工具** — 帮用户记录和提醒物品到期

### 未来（v3.0）
**AI 美食生活管家** — 从"减少浪费"到"智能生活决策"

### 价值跃迁

```
v1.0 工具价值：省心（不用记）
       ↓
v2.0 决策价值：省力（帮你想"先吃什么"）
       ↓
v3.0 生活价值：省钱+健康+品质（全方位家庭管理）
```

---

## 二、为什么升级

### 2.1 用户需求延伸

保质期管理的下一步自然延伸是：
- **"快过期了怎么办"** → 菜谱推荐
- **"总是过期怎么办"** → 采购建议
- **"全家怎么协作"** → 家庭共享
- **"数据有什么用"** → 消费分析

### 2.2 商业化机会

| 功能 | 变现方式 |
|------|---------|
| AI 菜谱推荐 | 食材电商分佣 |
| 智能采购清单 | 生鲜平台对接 |
| 消费数据分析 | 数据报告订阅 |
| 家庭共享 | 会员付费 |

### 2.3 开发者战略价值

这个升级让产品从"通用工具"变成**"AI + 美食垂类"的完整案例**——直接匹配目标岗位（AI 美食内容产品 / 本地生活 AI / 创作者工具 PM）。

### 3.2 核心新增模块详解

#### 模块 1：AI 厨房助手 ⭐⭐⭐

**核心场景**："我有 X、Y、Z 快过期了，能做什么菜？"

**功能设计**：

| 功能 | 描述 | AI 价值 |
|------|------|---------|
| 智能菜谱推荐 | 基于即将过期食材，推荐 3-5 个菜谱 | 解决"决策疲劳" |
| 食材替代建议 | "没有葱可以用什么代替" | 降低做饭门槛 |
| 剩菜改造 | "昨天的米饭能做什么" | 减少浪费 |
| 营养搭配 | 结合家庭成员需求推荐 | 健康增值 |
| 菜谱步骤引导 | 语音/图文教学 | 体验提升 |

**技术方案**：
- DeepSeek API + 结构化 Prompt（食材列表 + 用户偏好 + 营养需求）
- 菜谱数据库（爬虫 + 人工校验，初期 500+ 菜谱）
- RAG：用户历史菜谱 + 评价反馈

**数据指标**：
- 菜谱推荐点击率 ≥ 40%
- 推荐菜谱实际烹饪率 ≥ 20%

---

#### 模块 2：AI 图片识别录入 ⭐⭐⭐

**核心场景**：拍照自动识别商品信息，30 秒完成录入

**功能设计**：

| 功能 | 描述 | 技术方案 |
|------|------|---------|
| 商品识别 | 拍照识别商品名称、类别 | 多模态模型（GPT-4V / 通义千问 VL） |
| 保质期识别 | 识别包装上的保质期信息 | OCR + 正则提取 |
| 条码扫描 | 扫条码查询商品数据库 | 条码 API + 商品库 |
| 发票/小票识别 | 拍小票批量录入购物清单 | OCR + 结构化解析 |

**实现路径**：
1. v2.0：接入多模态 API（硅基流动 / 通义千问 VL），单张图片识别
2. v2.5：条码扫描 + 商品库匹配
3. v3.0：小票批量识别

---

#### 模块 3：AI 消费分析 ⭐⭐

**核心场景**：月度报告告诉你"浪费了多少 + 怎么改善"

**功能设计**：

| 报告类型 | 内容 | 价值 |
|---------|------|------|
| 月度浪费报告 | 过期物品清单 + 金额统计 + 类别分布 | 量化浪费 |
| 采购建议 | 基于消耗率推荐采购量 | 减少过度采购 |
| 消费趋势 | 各品类消费曲线 + 同比环比 | 发现规律 |
| 省钱成就 | 累计节省金额 + 等效物品 | 正向激励 |

**可视化**：
- 仪表盘（浪费金额、节省金额、健康分数）
- 趋势图（按月/品类）
- 排行榜（最常过期 TOP 5）

---

#### 模块 4：家庭协作 ⭐⭐

**核心场景**：全家人共同管理，谁买的谁负责消耗

**功能设计**：

| 功能 | 描述 |
|------|------|
| 家庭组 | 创建家庭，邀请成员（最多 6 人） |
| 角色权限 | 主理人 / 成员 / 只读 |
| 消息通知 | "妈妈买了 5 盒酸奶" |
| 任务分配 | "这周谁负责消耗牛奶" |
| 共享日历 | 家庭采购日、清理日 |

**技术方案**：
- 从本地 JSON 迁移到云端（Supabase / Firebase）
- 实时同步（WebSocket）
- 离线优先 + 冲突合并

---

#### 模块 5：智能采购 ⭐

**核心场景**：AI 帮你生成购物清单，还能一键下单

**功能设计**：

| 功能 | 描述 |
|------|------|
| 自动采购清单 | 基于消耗速度 + 库存预警 |
| 智能比价 | 跨平台价格对比（京东/拼多多/美团） |
| 一键下单 | 跳转电商平台（分佣变现） |
| 囤货建议 | 促销期囤货推荐 |
