# 到期管家 · 产品总览

> AI 原生桌面应用 · 独立完成 0→1 产品全流程
> 更新：2026-07-08

---

## 产品定位

**一句话**：让用户"拍张照就完成录入，看一眼就知道该处理什么"的家庭到期管理工具。

**覆盖品类**：食品、化妆品、药品、保健品、证件卡券、会员订阅、保险合同

---

## 为什么做这个产品

市面上的到期管理方案——备忘录、便签、超市 App——要么只能管理单一品类（仅食品或仅药品），要么需要用户手动填写大量字段，设完一次就不想再管了。

缺口是：缺少一个**通用品类、低门槛录入、能帮用户做判断**的管理工具。

AI 在这里的作用不是"加一个聊天框"，而是**改造管理流程中的两个关键卡点**。

---

## 两大核心创新

### ① 拍照即录入（替代手动输入）

传统做法：逐一手动输入品名、到期日、类别等多个字段 → 用户嫌麻烦，懒得用 → 工具吃灰。

AI 改造：拍一张产品包装图 → AI 自动识别品名、保质期、品类，一键填入。

> **效果**：录入操作从"多个字段手动填写"降到"5 秒拍张照"。
>
> **实现**：DeepSeek 多模态 API（base64 编码 → JSON 结构化输出），temperature=0.3 低随机度保证识别稳定性。

### ② 决策式管理（替代机械列表）

传统做法：按到期日期排列，用户逐条自己翻看、自己判断"这个怎么办"。

AI 改造：LLM 综合分析过期天数 + 品类特征 + 使用频率 → 输出**个性化的处理优先级和建议**。

> **效果**：用户"看一眼就知道先处理什么"，不需要自己翻列表做判断。
>
> **实现**：3 套差异化 Prompt 策略（洞察/建议/对话），temperature 分层调参，多轮上下文管理。

### 两项改造的共性

- AI 不是噱头——**每项改造都有明确的产品逻辑**：录入耗时 → 拍照解决；判断费力 → AI 给结论
- 双级降级策略：有 Key → 调 API，无 Key → 本地规则分析。AI 是加分不是依赖

## 功能全景

### 基础管理

| 功能 | 说明 |
|------|------|
| 多品类管理 | 食品/药品/化妆品等 7 大预设品类 + 自定义 |
| 分组折叠 | 按类别分组，到期天数排序，标题红色角标 |
| 图片存档 | 缩略图预览 + 点击查看大图 |
| 主动提醒 | 启动弹窗 + 系统托盘定时检查 |
| 灵活筛选 | 关键词 + 日期范围（7/30/90 天/自定义） |
| CSV 导出 | 一键导出（utf-8-sig，Excel 兼容） |
| 空状态引导 | 首次使用友好提示页 |

### AI 功能

| AI 功能 | 模型温度 | 核心能力 | 定位 |
|---------|---------|---------|------|
| 多模态图片识别 | 0.3 | 拍照 → JSON 结构化商品信息 | 替代手动录入 |
| AI 洞察报告 | 0.7 | 4 维度 Markdown 报告（关注/建议/趋势/提醒） | 辅助决策 |
| AI 对话助手 | 0.5 | 多轮上下文 + 城市感知 + 记忆管理 | 自然语言交互 |
| AI 通用建议 | 0.7 | 分类场景给出处理建议 | 辅助决策 |

## 技术概要

| 维度 | 选型 | 选型理由 |
|------|------|---------|
| 语言/框架 | Python + PySide6 | 跨平台、原生体验、开发效率高 |
| AI 模型 | DeepSeek API（OpenAI 兼容协议） | 性价比高、支持多模态、兼容主流生态 |
| 数据存储 | 本地 JSON | 零服务器、数据完全私有、易备份 |
| 多线程 | QThread + Signal/Slot | 不阻塞 UI，4 个 Worker 并行 |
| 降级策略 | 双级（API 主路径 + 本地兜底） | 无网/无 Key 环境正常使用 |
| 代码规模 | ~3000 行 Python | 零外部运行时依赖（除 PySide6） |
| 跨平台 | macOS / Windows / Linux | PySide6 原生跨平台支持 |
