## AI Prompt 设计（核心学习点）

三个 AI 功能用同一个 DeepSeek API，但 Prompt 策略完全不同：

### AI 洞察（结构化分析）
- **输入**：完整物品清单（名称/类别/日期/状态）+ 用户城市 + AI 记忆
- **输出**：四段式结构（📌本周关注 / 💡处理建议 / 📈趋势发现 / ⚠️重要提醒）
- **核心**：把原始数据转化为行动建议，不是展示数据而是理解数据

### AI 助手（对话式）
- **输入**：对话历史（最近 20 条）+ 库存数据 + 城市位置 + AI 记忆
- **输出**：自由问答
- **核心**：system prompt 设定了 5 条规则（地理感知/上下文连贯/结构清晰/事实准确/简洁实用）

### 关键设计决策
- 温度参数：洞察 0.7（需要创意），助手 0.5（需要准确）
- 聊天历史窗口：20 条（防 Token 溢出）
- AI 记忆：用户手动写入持久信息，每轮对话自动注入 system prompt
- 无 Key 降级：显示本地到期数据，不显示空白页

## 竞品分析

### 差异化优势
1. **AI 深度集成** — 竞品（iOS 提醒事项、Google Keep、Notion 模板）都没有 AI 能力
2. **全品类覆盖** — 不只食品，覆盖证件/会员/保险/药品
3. **本地隐私** — 数据不上云

### 核心劣势
1. **无移动端** — 到期管理是移动场景，桌面版覆盖不足
2. **无推送通知** — 微信订阅消息也有单次授权限制，真正推送需要原生 App
3. **需自备 API Key** — 普通用户的门槛

## 自我纠错记录（重要学习点）

AI 自己的评审也需要被审查。以下是第一版评审中发现的事实错误和逻辑问题：

### 事实错误
1. **说 settings.json 在 .gitignore 里但又说「推到 GitHub 会泄漏」** — 自相矛盾，实际在 .gitignore 不会被推
2. **说「快速点击智能建议可能启动多个 Worker」** — 实际按钮点击后立刻 disabled，Qt 单线程不会出现这个问题
3. **AI 集成深度给了 9 分** — 实际是广度（3 个入口用同一个 API），不是深度。应给 6-7 分

### 逻辑问题
1. **综合 7.2 分没有计算依据** — 各项平均只有 6.8 左右
2. **竞品「保质期管家」未验证是否真实存在** — 不能编造竞品名字
3. **「聊天面板挤压主界面」** — 实际 QSplitter 可拖拽，720px 完全够用

### 纠正后的结论
- 综合评分从 7.2 修正为 6.8
- 线程安全从 6 分修正为 8 分
- 数据安全从 6 分修正为 7 分
- AI 集成深度从 9 分修正为 7 分

**教训**：AI 的输出必须逐条核实，即使是"专业评审"也可能包含事实错误和自相矛盾的论述。

## 关于平台选择的讨论

### 小程序 vs 桌面版

**结论：小程序是更合适的平台，但不急于现在做**

- 到期管理是移动场景（逛超市查库存、收到提醒点开处理）
- 微信小程序有相机、分享、扫码即用
- ⚠️ 但微信订阅消息 ≠ 推送通知（用户需主动授权，每次授权只发一条）
- ⚠️ 云开发免费额度有限，不是完全免费

### API Key 门槛的解决方案

| 阶段 | 方案 | 说明 |
|------|------|------|
| 现在 | 用户自带 Key | 零成本，适合学习和验证 |
| 下一步 | 后端代理（Flask/FastAPI） | 用户无感知，你承担费用（约 1 元/人/月） |
| 未来 | 免费额度 + 付费订阅 | 注册送 20 次，超出自费 |

### 建议路线
```
桌面版（验证 AI + Prompt）→ 后端服务（解决 Key 门槛）→ 小程序/Web（面向真实用户）
```

## 学到的 AI 产品经理经验

1. **数据 → Prompt → 洞察** 是 AI 产品核心链路，不是数据 → 图表 → 用户自己看
2. **同一个 API，不同的 Prompt = 完全不同的产品体验**
3. **降级方案是必须的** — 没 AI 的时候产品不能变空白页
4. **Prompt Engineering 的三个关键**：结构（分段输出）、约束（字数/格式/语气）、上下文（注入用户数据）
5. **AI 功能重叠是常见错误** — 智能建议和 AI 洞察看起来不同，但用户不知道点哪个
6. **评分要自我纠错** — 第一版评分中 AI 集成深度给了 9 分，实际应该是 7 分（广度≠深度）
7. **柱状图 vs 折线图** — 计数数据用柱状图，连续数据用折线图。折线插值会产生"1.3 个商品"这种无意义的值
8. **微信订阅消息 ≠ 推送通知** — 不能把订阅消息当成和原生推送一样的体验来宣传
